FRAIL's Use Cases

Didactics

FRAIL can be used for didactics purpose. Students who participate in classes involving FRAIL, can learn how to prototype NPC in multi-agent systems like computer games or how specific methods of controlling NPC works. Other purposes may involve conducting artificial intelligence experiments in real-time, multi-agent systems provided by FRAIL.

FRAIL was already used in didactics at Wroclaw University of Technology at faculty of Computer Science and Management - http://www.ii.pwr.edu.pl/. The main goal was to teach the students how to use planning methods in problem solving process and to introduce them to a small part of computer game development. Students had to get familiar with FRAIL, behaviour tree (BT), finite-state machine (FSM) and hierarchical task network (HTN) methods. Official FRAIL version, available on GitHUB - https://github.com/frailpwr/frail_pwr, was used in this exercise.

Details

Students had to create NPC which will be confronted with FRAIL's reference boss, using methods of controlling artificial characters available in FRAIL. The winner is the character which wins the duel by killing its opponent.

A special game map was created for exercise purposes. It includes dynamic, interactive objects like exploding barrels, medical kits, etc. and static objects which refers to map's sculpture, like pillars and walls.

Simple map scheme is presented above. Red dots refers to spawn points, violet dots are exploding barrels, light green dot presents medical kit position, dark green is a double damage buff spot, the cyan circle refers to healing pool, the biggest yellow circle is arena's available moving region and the black squares present pillars.

AI-controlled characters standing opposite each other try to kill their opponent using their unique tactics. Every character can dodge attacks, move all over the arena, attack using three different attack types, interact with environment. AI characters had knowledge of only a small part of the world state, which contained every data available in game. Based on the mentioned, partial data they had to execute appropriate action, according to author's design.

The whole exercise was separated into a few smaller tasks. First task realization was rewarded with 2 points and involved NPC implementation using finite-state machine method. Second task involved hierarchical task network or behaviour tree. Student could choose one method and create NPC according to it. The second task was rewarded with 4 points. Third task encouraged students to create new NPC actions, like new attacks, dodges, movement styles, etc. It was rewarded with 2 points. Last two tasks referred to students' reports in which they had to describe and examine methods they worked with. Those two tasks were rewarded 1 point each and concerned both used methods effectiveness.

Tournament

Although FRAIL provides a great way of acquiring knowledge it also brings some entertainment. It was already used for the tournament, announced at Wroclaw University of Technology at Faculty of Computer Science and Management in 2014.

Students who participated in exercise which concerned FRAIL, were allowed to join the competition. Those who wished to join the tournament, had to send an email message with zip archive file attached. The archive had to include NPC's XML logic model, NPC's documentation created by its author, *.cpp and *.h files referring to NPC code used in FRAIL and other essential files needed to use the NPC in FRAIL.

FRAIL's tournament was conducted using FRAIL in the special, stripped version available at GitHUB - https://github.com/frailpwr/frail_pwr/tree/tournament. This version provided only one, the official tournament map, the same which was used in exercise mentioned in Didactics section.

The FRAIL tournament's was prepared to emerge the best NPC. The tournament was an all-play-all tournament type, which involved duel between all submitted NPCs. All duels were recorded and are available on YouTube website - here.

The winner is the character which collected the highest score. Every duel lasted 2 minutes or ended earlier if one fighting NPC won. The score was calculated using simple formula:

score = dmgDone - dmgTaken + 1000 * win

Score is the equation's outcome which involves damage dealt to the enemy (dmgDone), damage taken from enemy's attacks (dmgTaken) and win flag which equals 1, when the duel was won or 0, when the duel was lost.

To make the tournament fair some restrictions had to be introduced. The restrictions concerned especially actions which affected final score and duel's gameplay. Default NPC configuration was:

  • Damage - melee: 30
  • Damage - fireball: 15
  • Damage - anger mode: 30
  • Maximum health points: 200
  • Initial health points: 200
  • Maximum movement speed: 0.05
  • Animation duration for fireball attack (ms): 550
  • Animation duration for melee attack (ms): 650
  • Animation duration for anger mode attack (ms): 1000
  • Minimum fireball attack action duration (ms): 600
  • Minimum melee attack action duration (ms): 1200
  • Minimum anger mode attack action duration (ms): 1200

Three attack types were available:

  • Near range attack (melee)
  • Long range attack (fireball)
  • High jump attack (anger mode)

Any other attacks were recognized as a violation. Every attack type had to be implemented with corresponding animation.

Result

Students submitted ten NPCs to the FRAIL tournament. Three of them who won 1st, 2nd and 3rd places were rewarded with extra points and computer games provided by Techland company.

The scoreboard is presented below.

Header row and column in the scoreboard represent NPCs with their names included as values of cells. Columns in their cells hold values equal to points collected/lost by NPC specified in the header cell. Values in rows represent collected/lost points by the enemy in reference to the current NPC, identified with name presented in header row.

NPCs with 0 score are NPCs which were banned from the tournament due to violations.

All NPCs, their source code, special version of FRAIL for tournament purpose are available on FRAIL's GitHUB repository in tournament branch. Duels were recorded and uploaded on YouTube.
